# CMZ_VIBRO
Разработка системы вибромониторинга



1) ПЕРЕВОД НА SPI (выполнено)

Используем двойную буферизацию для чтения и отправки данных. Как только заполнили один буфер отсчетами с датчика (1600) задача отправки принимает буфер, а задача чтения берет свободный буфер. Сейчас вместо отпрвки рассчет RMS и вывод значеняи в монитор порта.

Необходимо определиться, что используем в будуем: WebSocket или MQTT? 
Необхожимо реализовать формирование пакета для отправки и написать сервер на питоне (для теста на коленке)
На сервере необходимо:
   Отфильтровать сигнал (как минимум полосовой фильтр (10-1000 Гц) для устранения постоянной составляющей и ВЧ шумов или отсекать тренд)
   БПФ
   Интегрирвоание в частотной области для получения спектра виброскрости
   По спектру расчет RMS (вроде бы легче чем во временной)
   По желанию ОБФ дял графика 

   ВОПРОС А ВЫДЕРЖИТ ЛИ ВООБЩЕ СЕРВЕР И ОДНА ЕСП ОТПРАВКУ 9600 отсчетов каждую секунду и обработку?
