Данные корректно считываются с датчика и передаются в задачу отправки по сети. 
Однако на данный момент ни через websocket ни через mqtt не получается отправлять корректно ~38кб данных в секунду
Ошибка точно не связана с задачей чтения или мощностями esp (тестировал просто отпарвку бинарных данных ./beffers.c void test_mqtt(void *pvParameters) )
Первые N пакетов передаются в очередь на отправку (порядка 10-15 иногда меньше иногда больше), после чего очередь outbox переполняется т.к данные не успевюат отправиться (увелечение outbox не решит проблему, только отсрочит т.к в любом случае данные не успевают отправляться, если конечно пакет не ~>1кб, тогда все отлично и замечательно)
ВОПРОС Как именно отправлять данные на сервер?
Как кажется решение в увеличении времени между отправками, но тогда теряется возможность мониторинга в реальном времени спектра и RMS (врядли СКЗ виброскорости за 10 секунд дает корректную информацию, удары будут не заметны)
